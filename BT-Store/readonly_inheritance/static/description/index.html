Have you ever wanted to mark a whole notebook page or a whole group as read-only?

Here is the solution!

The easiest way to handle read-only attributes

It's no longer necessary to add read-only attributes to all fields in a section. You can set the read-only attribute on the parent element, and this module does the rest. 

<group readonly="1">
    ...
</group>

Works on all elements

You can set the read-only attribute on all elements you want. It works on forms, groups, pages and all other elements.

This module will pass the read-only attributes recursively to all sub elements in the same view definition. 

<page readonly="1">
    ...
</page>

Force an inner read-only attribute

If you want to force a read-only attribute from a sub element, you can set the options="{'force_this_readonly': True}" option on the element, and the read-only attribute of the current element will be passed to its sub elements.

With this, you can force 'True' or 'False' for the read-only attribute as well as force using the read-only attribute from the view definition on the current and on its sub elements. 

<page readonly="1">
    <group string="Sale" readonly="0" options="{'force_this_readonly': True}">
        <field name="property_payment_term_id"/>
        <field name="credit"/>
        <field name="trust" readonly="1" options="{'force_this_readonly': True}"/>
    </group>
    <group string="Purchase">
        ...
    </group>
    <group string="Fiscal Information">
        ...
    </group>
</page>

readonly="x" vs. attrs="{'readonly': x}"

It works with the readonly="x" attribute as well as with the attrs="{'readonly': x}" attribute. 

<page attrs="{'readonly': [('is_company', '=', False)]}">
    ...
</page>

All you need to know

All read-only attributes will be passed recursively to all sub elements

You can force an inner read-only attribute with:
options="{'force_this_readonly': True}"
